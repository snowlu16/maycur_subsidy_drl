# 每刻补贴脚本需求调研与排查框架

开发或修改补贴脚本前，**必须将以下核心问题与业务确认清楚**。只有当业务规则清晰、无歧义且覆盖所有边界情况时，编写出的 `.drl` 脚本才能准确无误。

---

## 1. 核心确认清单 (Requirements Checklist)

### 1.1 维度与编码
- [ ] **是否已在后台配置补贴标准？列维度是否为城市？**
  - **红线规则**：补贴标准中的列维度**必须是城市**。若不区分城市，也需在列上配置“全部城市”或“默认城市”的表维度。
  - 获取并记录 **补贴标准业务编码 (`bizCode`)**。
- [ ] **生成明细对应什么费用类型？**
  - 获取并记录 **费用类型编码 (`feeCode`)** (如差旅补贴 `"88.004"`，餐饮补贴 `"020.001"` 等)。

### 1.2 人员计算范围 (`Employee` vs `TravelPartner`)
- [ ] **报销人（提单人）是否享受补贴？** (多数情况享受，但个别场景如代提单或纯同行人报销时不给提单人计算)。
- [ ] **是否计算同行人（参与人）？**
  - 若包含同行人，是从行程组件上的 `TravelPartner` 获取，还是从单据头部的参与人获取？
  - 报销人与同行人生成的补贴明细是**独立每人一条**，还是**汇总为一条总额费用**？

### 1.3 汇总与生成规则 (Aggregation Mode)
请明确选择以下哪种生成逻辑：
1. **按天汇总 / 一天一条**：根据出差天数，每天生成一条单独的费用明细（费用消费日期即当天）。
2. **按行程汇总 / 一段行程一条**：针对每一段出差行程，合并计算该段行程所有天数的补贴金额，生成一条费用记录（费用消费日期为该段行程的开始~结束区间）。
3. **全单汇总 / 一张单一条**：将单据内所有行程的补贴金额求和，生成一条总计费用。

### 1.4 申请单挂靠规则
- [ ] **单据若为出差申请单，其明细模式是哪种？**
  - 若为 **“需要在行程下填写申请明细”**，则所有生成的补贴记录必须通过 `setTravelRouteCode(...)` 挂靠至特定行程 ID 之下，否则在单据详情页不显示。

## 2. 特殊与高级场景确认 (Advanced/Edge Cases)

### 2.1 半天与时间阈值 (Half-Day Cutoff)
- [ ] **是否有半天计算规则？**
  - 通常以 `12:00` 为界（可配置）：
    - 出发当天：出发时间 `< 12:00` 算全天；`>= 12:00` 算半天。
    - 返回当天：返回时间 `<= 12:00` 算半天；`> 12:00` 算全天。
  - 半天额度是全天额度的 50%，还是单独在标准表中配半天的具体数字？

### 2.2 行程重叠与重复提单去重
- [ ] **如果同一个单据内有多段行程发生重叠（如 1.1~1.3 与 1.2~1.4），重叠部分怎么算？**
  - 通常将同一员工在多段行程中的日期抽取存入全局 `dateMap` 进行去重（`allowanceService.isDayHalveOrOverlap(...) == -1` 时自动剔除重叠日）。
- [ ] **跨单据之间的重叠行程如何校验？**
  - 建议前端配置开启：**智能审核 -> 日期重复费用校验**。

### 2.3 费用与事件扣减规则 (Deduction Rules)
- [ ] **单据内出现特定费用或勾选特定选项时，是否扣减补贴？**
  - *场景 1（发生特定费用扣减）*：若报销单中报销了“酒店费/招待费”，则当天的补贴金额减半或者扣为 0。需明确：是单笔扣减一次，还是只要存在就当天全扣？
  - *场景 2（行程选项联动扣减）*：行程组件上新增了自定义选填字段（如“是否公司派车”、“是否提供三餐”），若选择了“是”，则减免对应比例的交通/餐饮补贴。

### 2.4 常驻地校验 (Residence Location Check)
- [ ] **若出差目的地属于员工的常驻地，是否给予补贴？**
  - 许多企业规定：出差目的城市若属于员工档案中维护的第一/第二常驻地（如长期驻派或同城跨厂区），则不发放差旅补贴（金额为 0）。
  - 需确认匹配级别：精确到 **省** 匹配、**市** 匹配还是 **区县** 匹配？

### 2.5 差额补贴与节约补贴
- [ ] **差额返还**：如设定住宿上限为 300 元，员工实际报销 220 元，是否将省下的 80 元（或按 80% 比例 = 64 元）作为“节约补贴/差额补贴”发放？

## 3. 客户调研问卷提单模板 (Customer Template)

在给集成团队提工单或同客户需求对接时，请直接复制并填写以下表格：

```markdown
### 【差旅补贴规则确认表】

1. **基本编码说明**
   - 补贴标准业务编码 (`bizCode`)：__________________
   - 补贴生成费用类型编码 (`feeCode`)：__________________

2. **生成频率与汇总格式**
   - [ ] 按天生成（每人每天一条）
   - [ ] 按行程生成（一段行程每人一条）
   - [ ] 按行程汇总所有人（一段行程报销人+同行人合为一条）

3. **人员范围**
   - 是否包含提单报销人：[ ] 是  [ ] 否
   - 是否包含行程同行人：[ ] 是  [ ] 否

4. **半天与重叠逻辑**
   - 是否以 12:00 为界计算半天：[ ] 是（半天减半）  [ ] 否（全天）
   - 多段连续/重叠行程合并去重：[ ] 是  [ ] 否

5. **扣减与常驻地排查**
   - 常驻地出差不给补贴：[ ] 是（匹配至：省/市/县）  [ ] 否
   - 关联费用扣减（如报销餐饮费则当天补贴减半）：描述规则：__________________
   - 关联表单自定义选项扣减（如选公司派车补贴为0）：描述选项与规则：__________________
```
